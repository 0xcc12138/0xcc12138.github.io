UNICODE_STRING

简介

在Windows内核中，UNICODE_STRING结构体用于表示一个以UNICODE编码的字符串。它在内核和用户模式下的结构相同，并在内核驱动编程中用于处理字符串操作。

Length：

  • 类型：USHORT
  • 描述：表示字符串的实际长度，以字节为单位，包含字符串内容但不包括终止的空字符（即不一定为0终止）。Length值的上限是MaximumLength。

MaximumLength：

  • 类型：USHORT
  • 描述：表示分配给字符串缓冲区的最大长度（以字节为单位）。MaximumLength通常大于或等于Length，以确保在需要时能扩展字符串。
  • 在分配时通常包含一个额外的2字节空间以便于0结尾的Unicode字符串处理，但这取决于特定的使用场景。

Buffer：

  • 类型：PWSTR（指向WCHAR类型的指针）
  • 描述：这是指向字符串实际内容的指针。每个字符使用2个字节（宽字符WCHAR）存储。
  • Buffer在Length和MaximumLength的限制下保存字符串内容。

初始化的方法

1.最常用的方法是使用 RtlInitUnicodeString，它直接将字符串赋给 UNICODE_STRING。

参数：第一个参数是指向 UNICODE_STRING 的指针，第二个参数是常量 Unicode 字符串指针。

特点：无需手动设置 Length 和 MaximumLength，直接初始化为字符串内容的长度。

2.使用 RtlInitEmptyUnicodeString

如果想初始化一个空的 UNICODE_STRING，可以使用 RtlInitEmptyUnicodeString，手动指定 Buffer 缓冲区。

参数：第一个是 UNICODE_STRING 指针，第二个是缓冲区指针，第三个是缓冲区大小（字节）。

特点：适合预先分配好缓冲区，且内容需后续填充的情况。

常用API

ZwCreateFile

  • 参数简述：
    • OUT PHANDLE FileHandle：指向一个句柄变量的指针，用于接收创建或打开后的文件句柄。
    • IN ACCESS_MASK DesiredAccess：指定对文件的访问权限，如GENERIC_READ（读访问）、GENERIC_WRITE（写访问）等。
    • IN POBJECT_ATTRIBUTES ObjectAttributes：一个指向OBJECT_ATTRIBUTES结构的指针，用于描述文件对象的属性，包括文件名、对象名等。
    • OUT PIO_STATUS_BLOCK IoStatusBlock：指向一个IO_STATUS_BLOCK结构的指针，用于返回操作的状态信息，如完成状态、传输字节数等。
    • IN PLARGE_INTEGER AllocateSize：指定文件的初始分配大小（可选参数）。
    • IN ULONG FileAttributes：指定文件的属性，如FILE_ATTRIBUTE_NORMAL等。
    • IN ULONG ShareAccess：指定文件的共享访问模式，如FILE_SHARE_READ（允许其他进程读共享）等。
    • IN ULONG CreateDisposition：指定创建文件的处置方式，如FILE_CREATE（创建新文件）、FILE_OPEN（打开现有文件）等。
    • IN ULONG CreateOptions：指定创建文件的选项，如FILE_NON_DIRECTORY_FILE（不是目录文件）等。
    • IN PVOID EaBuffer：指向扩展属性（EA）缓冲区的指针（可选参数）。
    • IN ULONG EaLength：扩展属性缓冲区的长度（可选参数）。

一些小知识

扇区（Sector）：是磁盘存储数据的基本物理单位。在传统的磁盘存储中，扇区大小通常是固定的，一般为 512 字节。这是磁盘硬件设计层面上划分数据存储区域的最小单元。当磁盘进行数据读写操作时，数据的传输是以扇区为单位进行的。例如，在读取一个文件时，磁盘会将包含文件数据的扇区内容读取出来。

簇（Cluster）：是操作系统用于管理磁盘空间的逻辑单位。它是由一个或多个扇区组成的。簇的大小不是固定的，它取决于文件系统的格式化设置和磁盘的容量等因素。例如，在 FAT32 文件系统中，簇大小可能是 4KB（即 8 个 512 字节的扇区）。

注册表

注册表数据类型

REG_SZ（字符串值）

定义和用途：这是最常见的注册表数据类型之一。REG_SZ 代表一个以空字符（’\0’）结尾的字符串。它用于存储简单的文本信息，如文件路径、名称、描述等。例如，在 Windows 操作系统中，许多应用程序会将自己的安装路径存储在注册表的某个键下，以 REG_SZ 类型的字符串值表示。

REG_BINARY（二进制值）

REG_BINARY 用于存储二进制数据。这种数据类型可以包含任何形式的二进制信息，如设备驱动程序的配置数据、加密密钥、自定义的二进制格式数据等。由于其能够存储复杂的非文本数据，在一些需要存储特定格式数据（如硬件设备的设置信息）的情况下非常有用。

REG_DWORD（双字值）

REG_DWORD 是一个 32 位（4 字节）的无符号整数类型。它用于存储整数值，常用于表示一些简单的数字选项或状态信息。例如，可以用 REG_DWORD 来存储一个布尔值（0 表示假，非 0 表示真），或者一个计数、版本号等数字信息。

REG_QWORD（四字值）

REG_QWORD 是一个 64 位（8 字节）的无符号整数类型。它类似于 REG_DWORD，但可以存储更大范围的整数。在处理需要 64 位整数的情况（如文件大小的表示、高精度的时间戳等）时会使用 REG_QWORD。

REG_MULTI_SZ（多字符串值）

REG_MULTI_SZ 用于存储多个以空字符（’\0’）结尾的字符串。这些字符串之间也以两个连续的空字符（’\0\0’）分隔。它通常用于存储一组相关的字符串，如列表、路径数组等。

REG_EXPAND_SZ（可扩展字符串值）

REG_EXPAND_SZ 也是一种字符串类型，但它与 REG_SZ 的区别在于其中可以包含环境变量引用。当系统读取这种类型的值时，会自动将其中的环境变量引用展开为实际的值。这对于存储需要根据环境变量动态变化的路径等信息非常有用。

ZwDuplicateObject

ZwDuplicateObject 是 Windows 内核模式下的未文档化 API，主要用于复制一个句柄到另一个进程或当前进程。它允许开发人员在不同进程之间共享对象的句柄，同时控制对这些对象的访问。以下是该 API 的参数介绍和使用示例。

  1. SourceProcessHandle:
    • 类型：HANDLE
    • 指向源进程的句柄，该进程拥有 SourceHandle。必须具有对源进程的访问权限。
  2. SourceHandle:
    • 类型：HANDLE
    • 需要复制的源句柄。此句柄可以是对任何对象（如文件、事件、互斥体等）的句柄。
  3. TargetProcessHandle:
    • 类型：HANDLE
    • 指向目标进程的句柄。该进程将获得复制的句柄。可以是当前进程的句柄（即 NtCurrentProcess()）。
  4. TargetHandle:
    • 类型：PHANDLE
    • 指向接收复制句柄的变量的指针。调用成功后，此变量将包含目标句柄。
  5. DesiredAccess:
    • 类型：ACCESS_MASK
    • 目标句柄的访问权限。指示调用方希望在目标进程中获得的权限。
  6. HandleAttributes:
    • 类型：ULONG
    • 指定句柄的属性，如 HANDLE_FLAG_INHERIT 或 HANDLE_FLAG_PROTECT_FROM_CLOSE 等。
  7. Options:
    • 类型：ULONG
    • 指定复制操作的选项。可用的选项包括：
      • DUPLICATE_CLOSE_SOURCE: 在复制成功后关闭源句柄。
      • DUPLICATE_SAME_ACCESS: 允许访问权限在目标句柄中与源句柄相同。

返回值

ZwDuplicateObject 返回一个 NT 状态码，表示操作的成功与否。如果成功，返回 STATUS_SUCCESS，否则返回相应的错误代码。
