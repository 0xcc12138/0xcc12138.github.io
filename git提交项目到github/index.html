修改分支

有时候我们需要更新一个项目，可能是添加功能，也可能是修复一个Bug

这时候可能有一个需求就是，我希望原来的main分支变为old-main分支，然后换一个新的main上去

然后把原来的main改为v1.1

以下是步骤

1.首先初始化git

2.添加远程仓库 (这里是举个例子)

3.获取远程仓库的信息

4.查看目前记录的所有分支，此时应该要能看到远程仓库的

5.创建一个分支 #这里是创建一个main分支

6.切换到main分支 #git branch可以看目前所在的分支

7.拉取远程的main分支到现在所在的分支,也就是本地的main

8.重命名本地分支 #把main重命名为v1.1

9.推送v1.1到仓库，这样会多一个分支叫v1.1

10.在github的setting修改默认分支（如果需要，因为不能删除默认分支）

11.删除原来的分支main

现在有两个版本啦~
